Your work is organized into **workspaces** and **projects**, and surfaced as a
**timeline** that mixes human entries with AI-agent sessions. This page covers
how to move around that structure.

## Switch workspaces

You can belong to several workspaces. Use the **workspace switcher** at the top
of the sidebar to change the active one — the navigation, timeline, and filters
all update to that workspace.

## The dashboard timeline

The workspace dashboard shows a single timeline of recent activity — your work
entries and agent sessions together, newest first — with summary stats for the
selected period.

- **Period selector** — switch the range the dashboard summarizes (for example
  this week or this month).
- **Filters** — narrow the list by project, member, or tag.
- **Stats** — totals for the selected period.

## Project pages

Open a project from the workspace navigation to see only that project's
entries, with the same filtering. Projects have a name, a color, and a status
(active or archived); archived projects are hidden from pickers but keep their
history.

## What you can see

Visibility follows workspace and project membership:

- You always see **your own** entries.
- **Project members** see each other's entries within that project.
- **Workspace admins** can read all entries in the workspace.

Editing stays with the author — seeing an entry doesn't mean you can change it.
For the full role-by-resource model, see the
[Admin Guide](/workspace-admin/).

## Agent activity

AI-agent sessions appear on the same timeline. To dig into one agent's sessions,
token usage, and per-project breakdown, open its activity page — see
[Capturing agent activity](/guides/capturing-agents/).
