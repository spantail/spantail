---
title: 管理者ガイドの概要
description: Spantail の管理を担う人と、本ガイドが扱う設定への入り口。
---

このガイドは、Spantail のインスタンスとワークスペースを運用する人のためのものです。デプロイ
全体を管理する**インスタンス管理者**と、単一のワークスペースとそのプロジェクト・メンバーを
管理する**ワークスペースのオーナー/管理者**が対象です。

ここで扱う操作はすべて **Settings(設定)** ハブから行います。日々の利用 — 作業の記録・
タイムラインの閲覧・レポートの作成 — は[ユーザーガイド](/ja/guides/)に、インスタンスの
インストールとデプロイは[セットアップガイド](/ja/self-hosting/)にあります。

## 誰が何を管理できるか

Spantail は**インスタンス**管理と**ワークスペース**管理を分けています。1 人が両方を兼ねる
こともできます。

| ロール | 由来 | 管理対象 |
|---|---|---|
| **インスタンス管理者** | 最初にサインアップしたユーザー、および後から付与された人 | ユーザー・メール・ソーシャルログイン・エージェント機能、そして任意のワークスペースの設定/プロジェクト/メンバー。インスタンススコープ。 |
| **テンプレート作成者** | ユーザー単位の権限 | インスタンス管理者でなくても、インスタンス共通のレポートテンプレートを管理。 |
| **ワークスペースオーナー** | そのワークスペースを作成したユーザー | 管理者ができることすべて。削除されず、最後の管理者を割り込んで降格させることもできない。 |
| **ワークスペース管理者** | ワークスペースのロール | そのワークスペースの設定・プロジェクト・メンバー。 |
| **ワークスペースメンバー** | ワークスペースのロール | 管理操作は不可。ワークスペースの閲覧と作業の記録のみ。 |

管理者が管理するのは**コンテナ**(ワークスペース・プロジェクト・メンバー・インスタンス設定)
であって、ユーザー個人のデータではありません。レポート・API トークン・エージェント・アカウント
設定はセルフサービスで、インスタンス管理者であっても他人のレポートやトークンは編集できません。
秘匿情報は誰にも表示されません。

## 設定への入り口

サイドバー下部に固定された **Settings(歯車)** を開きます。Settings ハブは、左サブナビで各管理
セクションをまとめています。

- **ワークスペース** — 一般・プロジェクト・メンバー(ワークスペースのオーナー/管理者)
- **アカウント** — 自分のプロフィール・パスワード・API トークン・環境設定(セルフサービス)
- **レポート** — レポートテンプレート(インスタンス管理者またはテンプレート作成者)
- **システム** — ユーザー管理・AI エージェント・メール・ソーシャルログイン(インスタンス管理者のみ)

:::note[スクリーンショット]
*プレースホルダー — 左サブナビにワークスペース・アカウント・レポート・システムの各グループが
並んだ Settings ハブ。*
:::

## このガイドの内容

**ワークスペース管理** — 単一のワークスペースを管理する:

- [ワークスペース設定](/ja/admin/workspace-settings) — 名前・URL・ロゴ
- [プロジェクト](/ja/admin/projects) — プロジェクトの作成・編集・アーカイブ・メンバー編成
- [メンバーとロール](/ja/admin/members) — メンバーの追加とロール設定

**インスタンス管理** — デプロイ全体を管理する(インスタンス管理者):

- [ユーザー管理](/ja/admin/users) — ユーザーの作成・招待と権限付与
- [システム設定](/ja/admin/system-settings) — メール・ソーシャルログイン・エージェント機能
- [レポートテンプレート](/ja/admin/report-templates) — インスタンス共通のレポート書式の作成

初回デプロイ — Cloudflare のプロビジョニング、秘匿情報の設定、最初の管理者の作成 — は
[セットアップガイド](/ja/self-hosting/)で別途扱います。
