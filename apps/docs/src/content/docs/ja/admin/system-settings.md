Settings の**システム**グループには、インスタンス全体のスイッチがまとまっています。ここの画面は
**システム情報(About)**を除き、すべて**インスタンス管理者のみ**です。システム情報は全員に表示
されます。

これらの機能が依存する資格情報 — メール(SMTP/Email Routing)や OAuth のクライアントシークレット —
は、この UI ではなくデプロイ時に環境シークレットとして一度だけ設定します。セットアップガイドの
[設定](/ja/self-hosting/configuration)を参照してください。

## メール

**Settings → システム → メール。** Spantail がメール(招待・パスワードリセット・レポート配信)を
送信するかどうかを制御します。

- **メールを有効化** — 配信のオン/オフ。
- **送信元アドレス** — 送信者アドレス(配信の有効化に必須)。
- **送信元名** — 任意の分かりやすい送信者名。

送信元アドレスなしにメールを有効化することはできません — このガードにより、招待が送信時に失敗する
のを防ぎます。メールが**オフ**のときは、ユーザーのオンボーディングが、招待メールの代わりに一時
パスワード付きの直接作成に切り替わります([ユーザー管理](/ja/admin/users#ユーザーを追加する)を参照)。

:::note[スクリーンショット]
*プレースホルダー — 有効化トグルと送信元アドレス/名フィールドを備えたメール設定ページ。*
:::

## ソーシャルログイン

**Settings → システム → ソーシャルログイン。** メールとパスワードに加えて、**Google** または
**GitHub** でのサインインを可能にします。

- **プロバイダ別トグル** — Google と GitHub を個別に有効化。プロバイダは資格情報が環境に存在する
  場合にのみ有効化でき、それまではトグルが無効のままヒントが表示されます。
- **許可するメールドメイン(Google)** — 任意で、Google サインインを 1 つ以上のドメインに制限
  (カンマまたは改行区切り)。空にするとすべての Google アカウントを許可します。

:::note[スクリーンショット]
*プレースホルダー — Google と GitHub のトグルと許可ドメインフィールドを備えたソーシャルログイン
ページ。*
:::

ソーシャルログインは、インスタンスに最初の管理者ができるまで利用できません — セットアップ前に
ソーシャルサインインでインスタンスを乗っ取られないようにする意図的な安全策です。ブートストラップ
の規則は[セキュリティ](/ja/self-hosting/security)を参照してください。

## AI エージェント

**Settings → システム → AI エージェント。** エージェント活動機能の単一スイッチです。

- **オン** — ユーザーはエージェントを作成でき、**アカウント → エージェント**画面が表示されるため、
  AI エージェントが活動を span として取り込めます。
- **オフ** — エージェント UI は非表示になり、新しいエージェントは作成できません。

:::note[スクリーンショット]
*プレースホルダー — 機能トグルを備えた AI エージェントページ。*
:::

## リアルタイム更新

**Settings → システム → リアルタイム更新。** Server-Sent Events によるライブ更新の単一スイッチ
です。

- **オン** — 開いているブラウザタブに、変更（新しいエントリ・プロジェクト・レポート・メッセージ）
  が再読み込みなしで即時反映されます。
- **オフ**（既定） — タブがフォーカスを取り戻したタイミングで画面は再取得されます。その間のプッシュ
  はありません。

このスイッチがあるのは、接続中のタブごとにユーザー単位の Durable Object が常駐し、その実行時間が
Cloudflare アカウントの 1 日のクォータを消費するためです — **Workers Free プラン**では数人の
アクティブユーザーだけで数時間のうちに使い切ることがあります。Workers Paid プラン、または利用者が
ごく少ない Free プランのインスタンスで有効化してください。オフへの切り替えは新規接続から適用され、
すでに開いているタブは再読み込みまでストリームを保持します。

:::note[スクリーンショット]
*プレースホルダー — 機能トグルを備えたリアルタイム更新ページ。*
:::

## システム情報(About)

**Settings → システム → システム情報。** System の他の画面と違い、**このページはすべてのユーザーに
表示**されます(インスタンス管理者に限りません) — 誰でも稼働中のバージョンを確認できます。

製品名・稼働中の**バージョン**(対応する GitHub リリースにリンク)・**著作権**を表示します。ここに
設定項目はありません。

:::note[スクリーンショット]
*プレースホルダー — 製品名・バージョン・著作権を表示したシステム情報ページ。*
:::
