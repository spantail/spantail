# R2 assets (demo-ja)

Avatars and workspace logos for the `demo-ja` dataset, laid out to mirror the R2
uploads bucket 1:1:

- `avatars/<userId>` — one per user
- `workspaces/<workspaceId>/logo` — one per workspace

Files are **extensionless** (R2 object keys have no extension) and all **WebP**.
The filenames are the seed's deterministic ids, so their presence wires the
avatar/logo into `pnpm db:seed` automatically.

## Provenance / license

Placeholder artwork generated by `pnpm generate-avatars demo-ja`
([`packages/db/seed/generate-avatars.ts`](../../../packages/db/seed/generate-avatars.ts)):
a stylized silhouette per user and a monogram per workspace. No third-party
assets, no real people — free to redistribute. Swap any file for a real photo
without touching code; only its object key matters.

## Syncing to a bucket

`pnpm db:seed` uploads these to the local bucket automatically. For a remote
instance, sync this directory — the files are extensionless, so set the type
explicitly:

```bash
aws s3 sync examples/demo-ja/r2 s3://<bucket> --content-type image/webp
```

See [`packages/db/seed/README.md`](../../../packages/db/seed/README.md#r2-assets)
for the full remote-seeding flow.
